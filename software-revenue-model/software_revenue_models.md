# Software Revenue Models -- Summary

## 1. Subscription Model (Most Common ⭐)

### Description

Customers pay a recurring fee (monthly or yearly) to use the software.

### Examples

-   SaaS platforms (CRM, project management tools)
-   Cloud services

### Pricing Methods

-   Per user (e.g., \$10/user/month)
-   Tiered plans (Basic / Pro / Enterprise)
-   Usage-based (API calls, storage)

### Pros

-   Predictable recurring revenue (MRR / ARR)
-   Strong customer retention
-   Scalable

### Cons

-   Requires continuous value delivery
-   Customer churn risk

------------------------------------------------------------------------

## 2. One-Time License (Perpetual License)

### Description

Customers pay once to own the software indefinitely.

### Pros

-   Immediate large revenue
-   Simple pricing

### Cons

-   No recurring income
-   Harder to scale

------------------------------------------------------------------------

## 3. Maintenance & Support Fees

### Description

Ongoing fees for updates and support (15%--25% yearly).

------------------------------------------------------------------------

## 4. Usage-Based Model

### Description

Pay based on actual usage (API, storage, compute).

------------------------------------------------------------------------

## 5. Freemium Model

### Description

Free basic version + paid advanced features.

------------------------------------------------------------------------

## 6. Custom Development

### Description

Charge per project or per hour.

------------------------------------------------------------------------

## 7. Hybrid Model (Enterprise ⭐)

### Description

Combination of: - License - Maintenance - Custom development -
Subscription

------------------------------------------------------------------------

# Final Takeaways

-   Most common today: **Subscription (SaaS)**
-   Enterprise: **License + Maintenance + Custom**
-   Best strategy: **Hybrid model**
